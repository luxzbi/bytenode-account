bytenode 계정 · 개발자 / OAuth 설정

bytenode 통합 계정(SSO)을 내 서비스에 연동하는 방법입니다. 계정 원본은 bytenode이며, 이 서버가 로그인 UI와 OAuth 2.0 인가를 제공합니다.

1. 등록된 클라이언트

현재 등록된 서비스와 허용된 redirect origin입니다. (localhost는 항상 허용)

client_id허용 origin

새 서비스 등록 또는 origin 추가는 Vercel 환경변수 EXTRA_ORIGINS(형식: client_id|https://도메인, 쉼표 구분)로 합니다.

2. 간단 연동 (SSO 리다이렉트)

가장 쉬운 방법. 로그인/가입 버튼을 아래 주소로 연결하면, 완료 후 redirect 주소로 #bn_token=<토큰>을 붙여 돌아갑니다.

로그인:   https://bytenode-account.vercel.app/login?redirect=<내 서비스 주소>
회원가입: https://bytenode-account.vercel.app/welcome?redirect=<내 서비스 주소>

돌아온 페이지에서 토큰 저장:

const m = location.hash.match(/bn_token=([^&]+)/);
if (m) {
  localStorage.setItem('bn_token', decodeURIComponent(m[1]));
  history.replaceState(null, '', location.pathname);
}

3. OAuth 2.0 (authorization code)

표준 코드 교환 방식이 필요하면 아래 엔드포인트를 사용합니다.

엔드포인트설명
GET /authorizeresponse_type=code&client_id&redirect_uri&state — 로그인 후 redirect_uri?code=…&state=…로 이동
POST /token{ grant_type:"authorization_code", code, client_id } → { access_token, token_type, user }
GET /userinfo헤더 Authorization: Bearer <access_token> → 유저 정보
  • 인가 코드는 2분 내 1회 교환해야 합니다.
  • access_token은 bytenode JWT(30일)로, bytenode API에도 그대로 사용할 수 있습니다.
  • 토큰 검증은 항상 /userinfo 호출로 하세요 (직접 decode 신뢰 금지).

5. 계정 API (프록시)

엔드포인트설명
POST /api/login{ username, password } → { token, user }
POST /api/register{ username, displayName, password } → { token, user }
GET /api/meBearer 토큰 검증 + 유저 정보

이 서버는 CORS가 열려 있어 브라우저에서 직접 호출할 수 있습니다. 정책: 이용약관 · 개인정보처리방침
